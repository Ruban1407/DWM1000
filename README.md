# Goal of the Project

The project measures the **distance between two DW1000 modules** using **Single-Sided Two-Way Ranging (SS-TWR)**.

There are two devices:

```text
Initiator (Tag)             Responder (Anchor)

STM32 + DW1000  <------->  STM32 + DW1000
```

The **Initiator** starts the ranging process.

The **Responder** only replies.

---

# Overall Flow

```text
Power ON
    │
    ▼
Initialize STM32 SPI
    │
    ▼
Initialize DW1000
    │
    ▼
Configure DW1000 Registers
    │
    ▼
Send POLL packet
    │
    ▼
Responder receives POLL
    │
    ▼
Responder sends RESPONSE packet
    │
    ▼
Initiator receives RESPONSE
    │
    ▼
Read hardware timestamps
    │
    ▼
Calculate Time of Flight
    │
    ▼
Calculate Distance
```

---

# Step 1: Initialize the DW1000

The code first configures the DW1000.

It writes registers such as:

* Channel
* Data rate
* Preamble length
* TX power
* RF settings
* Antenna delay

After initialization, the DW1000 is ready to transmit and receive UWB packets.

---

# Step 2: Send a POLL Packet

The Initiator creates a small packet like:

```text
Frame Type
Sequence Number
Source Address
Destination Address
Function Code = POLL
```

The STM32 writes this packet into the DW1000's **TX buffer**.

Then it tells the DW1000:

> **"Start transmitting."**

From this point, **the DW1000 hardware** generates the UWB radio signal and transmits it. The STM32 is no longer involved in the actual RF transmission.

---

# Step 3: DW1000 Saves the TX Timestamp

The moment transmission begins, the DW1000 hardware automatically records the exact transmission time.

```text
TX Timestamp = T1
```

The STM32 does **not** measure this time.

The DW1000 hardware stores it in an internal timestamp register.

---

# Step 4: Responder Receives the Packet

The responder's DW1000 is continuously listening.

When it detects the incoming UWB frame:

* It stores the received packet.
* It records the exact reception time.

```text
RX Timestamp = T2
```

Again, this timestamp is generated by the DW1000 hardware.

---

# Step 5: Responder Sends a RESPONSE

The responder waits a known processing time.

Then it creates a response packet containing information such as:

* Poll reception timestamp (T2)
* Response transmission timestamp (T3)

It tells its DW1000 to transmit.

The DW1000 records:

```text
TX Timestamp = T3
```

---

# Step 6: Initiator Receives the RESPONSE

The initiator receives the response packet.

Its DW1000 automatically records:

```text
RX Timestamp = T4
```

Now the initiator has:

* Its own transmit time (T1)
* Its own receive time (T4)

And from the response packet it learns:

* Responder receive time (T2)
* Responder transmit time (T3)

---

# Step 7: Calculate Time of Flight

Now the software calculates how long the radio wave spent traveling through the air.

It computes:

```text
Round Trip Time = T4 - T1

Responder Processing Time = T3 - T2

Time of Flight = (Round Trip Time - Processing Time) / 2
```

This removes the time the responder spent processing the packet, leaving only the propagation time.

---

# Step 8: Convert Time to Distance

The code knows that radio waves travel at the speed of light:

```text
Distance = Time of Flight × Speed of Light
```

For example:

```text
Time of Flight = 33.4 ns

Distance

= 33.4 × 10⁻⁹ × 299,792,458

≈ 10 meters
```

---

# Where does the STM32 do the work?

The STM32 mainly performs control and calculations:

* Initializes the DW1000.
* Sends SPI commands.
* Builds packets.
* Reads timestamps from the DW1000.
* Applies the distance formula.

It **does not** generate or receive the UWB radio signal itself.

---

# Where does the DW1000 do the work?

The DW1000 hardware handles all the precise radio and timing operations:

* Generates the UWB pulses.
* Detects incoming UWB packets.
* Captures transmit timestamps.
* Captures receive timestamps.
* Performs very accurate timing measurements.

This hardware timestamping is what makes centimeter-level ranging possible.

---

# Summary of Responsibilities

```text
STM32
│
├── Configure DW1000
├── Send commands over SPI
├── Build POLL/RESPONSE packets
├── Read timestamps
└── Calculate distance

DW1000
│
├── Generate UWB signal
├── Receive UWB signal
├── Timestamp TX events
├── Timestamp RX events
└── Store timestamps in registers
```

So, in one sentence:

> **The STM32 controls the process and performs the math, while the DW1000 performs the precise radio transmission, reception, and timestamping. The distance is calculated by reading four hardware-generated timestamps and converting the resulting Time of Flight into meters.**
